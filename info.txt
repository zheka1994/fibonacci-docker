elastic beanstalk - это сервис оркестрации, предлагаемый Amazon Web Services для развертывания приложений, которые оркестрируют различные сервисы AWS, включая EC2, S3, Simple Notification Service, CloudWatch, автомасштабирование и Elastic Load Balancers.
RDS (Relational Database Service) - группа реляционных облачных СУБД в составе Amazon Web Services, позволяющая подписчикам заказывать экземпляры различных типов СУБД по подписной модели.
Делаются бекапы БД в случае проблем.
Amazon ElastiCache - о полностью управляемое хранилище данных в памяти и служба кэширования от Amazon Web Services (Например, отказоустойчивые Redis-кластеры)
VPC - virtual private cloud
Регионы - это различные дата центры, расположенные в разных частях света
VPC - это наши маленькие частные сети, изолированные моим аккаунтом
Создается одна VPC на весь мир
Если вбить в панель поиска VPC мы увидим нашу виртуальную сеть
Для взаиомдействия сервисов (наших приложений и БД-кластера и Redis кластера) должны настроить Security Group (Firewall rules)
Security Group по умолчанию созданы и позволяют пропускать входящий траффик из любой точки мира
С помощью Security Groups надо настроить, чтобы траффик на контейнер с приложением, на БД-кластер и на Redis
Security Group необходимо для разрешения входящего траффика на порт 80 с любого IP адреса, позволить трафик с порта <port> с IP-адреса <IP>

Services IAM - сервис для управления доступом к ресурсам AWS
Есть также замена docker hub от амазон AWS Elastic Container Registry ECR
Amazon Simple Storage Service (Amazon S3) – это сервис хранения объектов, предлагающий лучшие в отрасли показатели производительности, масштабируемости, доступности и безопасности данных.
